    循环里；老的 “0 / 1” 临时输入已关闭，录音/排队/重试都会驱动状态栏更新。改动在 main.py:20-335,
    src/keyboard/listener.py:9-273.
  - 新增 src/ui/status_bar.py：支持从 .env 读取 STATUS_ICON_* 路径加载 PNG/PDF 图标，默认仍用
    emoji；如果配置的是模板图标，可通过 STATUS_ICON_TEMPLATE=true 让 macOS 自动着色；排队数量会在图
    标旁附加数字。src/ui/__init__.py 是配套包文件。
  - .env.example:35-42 添加了图标相关示例配置，方便后续替换。

  使用 / 调整

  - 状态栏图标还是可以按住 ⌘（Command）拖动排序，和原生项目行为一致；如果无法拖动，确认鼠标先点住图
    标再按住 ⌘ 拖动。
  - 自定义图标：在 .env 中设置 STATUS_ICON_IDLE、STATUS_ICON_RECORDING、STATUS_ICON_PROCESSING、
    STATUS_ICON_RETRY 等路径指向 SVG/PNG/PDF（建议 18×18~22×22 的模板 PDF），重启即可生效。
  - 如需更多状态的独立图标，可继续补充同名环境变量（例如 STATUS_ICON_WARNING）；未配置的状态会回落
    到默认 emoji。
  - 由于状态栏按钮无法直接叠加徽章，自定义图片时队列数会作为按钮文本追加（或只显示在 tooltip 中）；
    若想做真正的描边/透明风格，可以提供模板图并保持 STATUS_ICON_TEMPLATE=true。
